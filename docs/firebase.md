# Firebase setup for NitaAi

## What you get

- OTP (phone number) authentication via Firebase Auth
- Firestore collections: `users`, `vendors`, `drivers`, `orders`
- Starter security rules and indexes in `firebase/`

## 1) Create a Firebase project

1. Firebase Console → Add project → create `nitaai-dev` (or your preferred name)
2. In Project settings, copy the **Project ID**
3. Update `.firebaserc` with your real project id (replace `nitaai-dev`)

## 2) Enable OTP authentication (Phone Auth)

Firebase Console → Authentication → Sign-in method:

- Enable **Phone**

Platform notes:

- Android: add your app + package name, then add **SHA-1/SHA-256** fingerprints in Project settings (required for phone auth on many setups)
- iOS: upload/configure **APNs** keys in Project settings
- Web: ensure your domain is allowed; Phone Auth uses reCAPTCHA on web

## 3) Create Firestore + deploy rules

1. Firebase Console → Firestore Database → Create database
2. From repo root, deploy rules and indexes:

   - `firebase deploy --only firestore`

Files:

- Rules: `firebase/firestore/firestore.rules`
- Indexes: `firebase/firestore/firestore.indexes.json`

## 4) (Optional) Storage + deploy rules

If you plan to store profile photos, invoices, etc.:

- `firebase deploy --only storage`

Rules file:

- `firebase/storage/storage.rules`

## 5) Run emulators locally (recommended)

From repo root:

- `firebase emulators:start`

Ports are configured in `firebase.json`.

## 6) Flutter integration (client)

This repo contains client-side Firebase code in `apps/nitaai_flutter/lib/`, but you still need a real Flutter app configuration:

1. In `apps/nitaai_flutter/`, run Flutter + FlutterFire setup:
   - `flutter pub get`
   - `dart pub global activate flutterfire_cli`
   - `flutterfire configure`
2. Replace `apps/nitaai_flutter/lib/firebase_options.dart` with the generated one
3. Add the platform config files generated by FlutterFire:
   - Android: `android/app/google-services.json`
   - iOS: `ios/Runner/GoogleService-Info.plist`

## Firestore schema (simple)

Firestore is “documents in collections”. Think of each collection as a table, and each document as one row.

### `users/{uid}`
One document per signed-in person (doc id = Firebase Auth `uid`).

Suggested fields:

- `uid`: same as document id
- `phoneNumber`: user’s phone
- `role`: `"user"` (or `"vendor"`, `"driver"`, `"admin"` for privileged accounts)
- `displayName`: optional
- `createdAt`, `updatedAt`: timestamps

### `vendors/{uid}`
One document per vendor account (doc id = vendor user’s `uid`).

Suggested fields:

- `vendorId`: same as document id
- `storeName`
- `phoneNumber`
- `address`: optional
- `isActive`: true/false
- `createdAt`, `updatedAt`

### `drivers/{uid}`
One document per driver account (doc id = driver user’s `uid`).

Suggested fields:

- `driverId`: same as document id
- `fullName`
- `phoneNumber`
- `vehicleNumber`: optional
- `isAvailable`: true/false
- `createdAt`, `updatedAt`

### `orders/{orderId}`
One document per order (doc id = auto-id).

Suggested fields:

- `orderId`: same as document id (copy stored for convenience)
- `userId`: customer `uid`
- `vendorId`: vendor `uid`
- `driverId`: driver `uid` (optional until assigned)
- `status`: `"placed" | "accepted" | "assigned" | "picked_up" | "delivered" | "cancelled"`
- `items`: list of `{ name, quantity, unitPrice }`
- `totalAmount`: number
- `deliveryAddress`: optional
- `notes`: optional
- `createdAt`, `updatedAt`

## Security model (high level)

- A signed-in user can read/write their own `users/{uid}` profile
- Vendors and drivers are expected to have their role set in `users/{uid}.role`
- An order can be read by the customer, the assigned vendor, the assigned driver, or an admin
